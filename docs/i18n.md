# Internacionalização da AURA

## Escopo atual

Idiomas suportados nesta fase:

- `pt-BR` como padrão
- `en-US` como secundário

## Infraestrutura

- Provider compartilhado em [`packages/core/src/i18n.tsx`](C:/dev/aura/packages/core/src/i18n.tsx)
- Persistência no banco em `professionals.locale`
- Persistência local como fallback com `aura.locale`
- Formatação por locale para data, data/hora e moeda

## Fluxos cobertos

- autenticação web
- shell principal do web
- dashboard web
- clientes web
- agenda web
- contratos web
- títulos e estados principais da inbox web
- resumo e timeline principais do detalhe da cliente no web
- shell global do mobile
- login mobile
- dashboard mobile
- clientes mobile
- agenda mobile
- notificações mobile
- resumo e núcleo da inbox da cliente no mobile

## Cobertura parcial intencional

Ainda existem textos operacionais menores em telas densas, especialmente em partes profundas do detalhe da cliente e em alguns formulários avançados do mobile, que permanecem em `pt-BR` por ora. A estratégia desta fase foi:

- cobrir infraestrutura
- cobrir fluxos principais
- evitar introduzir uma camada de i18n inconsistente ou excessivamente complexa

## Seletor de idioma

- web: seletor no sidebar/footer operacional
- mobile: seletor visível no hero da `ScreenShell`

## Regras

- novos textos de UI final devem entrar no dicionário compartilhado
- `pt-BR` permanece como fallback obrigatório
- chaves devem ser estáveis e orientadas a domínio/área
